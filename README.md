# Kondate Roulette (献立ルーレット)

日々の「今日のご飯、何にしよう？」という小さな悩みを、楽しさとAIの提案力でスピーディに解決するWebアプリケーションです。

# Features

### 1. 開発背景と本課題での実装範囲（選考課題：何を作ろうと思ったか / なぜか）
- **本課題での実装範囲:** 本作のベースとなるルーレットの基本機能や履歴保存の基盤は学校の授業課題で作成したものです。今回の二次面接事前課題（バイブコーディング課題）にあたり、**「日々の小さな不便をITで解決する」**というテーマに沿って、自ら**「Gemini AIを活用した献立提案機能」を独自に企画・ジャッジ・追加実装**しました。
- **解決したかった日常の不便:**
  毎日訪れる「今日のご飯、何にする？」という意思決定は、小さなことのようで意外と多くのエネルギーを使います。家族や同居人との間で発生しがちな「何でもいいよ」と言われるストレスや、メニューがマンネリ化してしまう日常の小さな摩擦を解決したいと考えました。

### 2. AI提案による不便の解消
メニューが思いつかない時、これまでは自分で絞り出す必要がありましたが、今回の拡張によって**「今の気分や大まかな種類、体調（例：さっぱりした和食、風邪気味で優しいもの）」**を入力するだけで、AIがその時のコンディションに最適なメニュージャンルを自動提案し、ルーレットの項目に即座に追加してくれる仕組みを実現しました。これにより、考える負担を最小限に抑えられます。

### 3. 生成AIとの協働とプロンプトの工夫（選考課題：AIとのやり取りや工夫）
短期間で機能拡張を行うにあたり、生成AI（Gemini/ChatGPT/Claude等）を単にコードを自動生成してもらう道具としてではなく、**「開発パートナー（シニアエンジニア）」**として位置づけ、以下の工夫を行いました。
- **徹底した事前すり合わせ:** すぐにコードを生成してもらうのではなく、まずは「何を解決したいか」「どのような仕様にすべきか」という**要件定義や、作りたい仕様の細かなすり合わせが終わってからコードを生成してもらう**アプローチを徹底しました。これにより、設計のブレや手戻りを防ぎました。
- **段階的な実装:** フロントエンドとDjangoバックエンドの結合、AI提案APIの組み込みなど、フェーズを小分けにして開発を進めました。
- **異なるAIツールの使い分け（マルチAIデバッグ）:**
  実装途中でエラーが発生し、1つのAIで解決が難しく詰まってしまった際には、**別のアカウントのAIや、キャラクター・特性の異なる種類のAI（ChatGPTやClaudeなど）へプロンプトを投げ直す**ことで、多角的な視点からアプローチを仰ぎ、トラブルを迅速に突破しました。
- **UI/UXの壁打ち:**
  「クリックしたときの手応え（ポチッと感）」や「立体感を出すためのシャドウ（影）」など、ユーザーが触って心地よいと感じるデザインや視覚的フィードバックの実現に向け、CSSの調整をAIと何度も壁打ちしながら組み立てました。

# Requirement

このアプリケーションを動作させるには、以下の環境およびライブラリが必要です。

* Python 3.10+
* Django 6.0
* django-environ 0.11.2 (環境変数管理用)
* SQLite3 (Django標準スタック)
